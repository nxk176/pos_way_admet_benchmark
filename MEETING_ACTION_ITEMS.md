## Non-Negotiable Quality Gates

- Every 3-property editing row must have exactly one input molecule, exactly three objective groups, and exactly two positive answers.
- Held-out splits must avoid molecule overlap with train for both inputs and answers.
- PubChem assay-local endpoints must not be described as curated target-level endpoints until enriched.
- BindingDB comparisons must stay within the same target and measurement type.
- Censored BindingDB values are retained for audit but excluded from ranking labels.
- Unknown agonist/antagonist state remains `modulation_numeric=0.5`; do not present it as measured neutral modulation.

## Commands

Regenerate the follow-up report:

```powershell
python .\scripts\generate_meeting_followup_report.py
```

Run a small ChEMBL traditional baseline smoke test:

```powershell
..\myenv311\Scripts\python.exe .\scripts\train_traditional_baselines.py edit-ranker `
  --data-dir .\data\chembl_3prop_2pos `
  --source-name ChEMBL `
  --max-train-queries 500 `
  --max-eval-queries 100 `
  --max-iter 10
```

Export ChEMBL LLM direct-edit prompts before calling any API:

```powershell
python .\scripts\run_llm_edit_baseline.py export-prompts `
  --data-dir .\data\chembl_3prop_2pos `
  --split test `
  --max-rows 20 `
  --few-shot-k 2 `
  --out .\reports\llm_edit_baselines\chembl_test_prompts.jsonl
```

The follow-up report command writes:

- `reports/meeting_followup_report.md`
- `reports/meeting_followup_report.json`

Traditional baseline training outputs go to `reports/traditional_baselines/`.
LLM baseline outputs go to `reports/llm_edit_baselines/`.
